## Acceptance Criteria

### AC1: QR Code Generation for Invite Links

**Priority:** High

**Scenario 1.1: Generate QR Code from Invite Link**
```
GIVEN a project administrator has generated an invitation link
WHEN the invite success modal is displayed
THEN a "Show QR Code" button is visible
AND clicking the button displays a QR code containing the invite URL
AND the QR code is scannable by mobile devices
```

**Scenario 1.2: Copy QR Code to Clipboard**
```
GIVEN a QR code is displayed in the invite modal
WHEN user clicks "Copy QR Code" button
THEN the QR code image is copied to clipboard
AND a success feedback "Copied!" is displayed
AND the image can be pasted into an email client
```

**Scenario 1.3: QR Code with Embedded Config**
```
GIVEN an invitation link with embedded Supabase config
WHEN QR code is generated
THEN the QR code contains the full URL with encoded config
AND scanning the QR code retrieves the complete configuration
```

**Verification Method:**
- Manual testing: Generate invite, display QR code, scan with mobile device
- Automated: Unit test for QR code generation with known URLs
- Regression: Ensure existing invite link functionality remains unchanged

---

### AC2: QR Code Scanner for Config Setup

**Priority:** High

**Scenario 2.1: Open QR Scanner Modal**
```
GIVEN user is on Supabase config setup page
WHEN user clicks "Scan QR Code" button
THEN QR scanner modal opens with camera preview
AND camera permission request is displayed (if not previously granted)
AND clear instructions explain the scanning process
```

**Scenario 2.2: Successful QR Scan**
```
GIVEN QR scanner modal is open with active camera
WHEN user positions valid invitation QR code in frame
THEN QR code is automatically detected and decoded
AND URL is extracted from QR code
AND Supabase config is decoded from URL
AND Form fields are auto-filled with decoded config
AND Scanner modal closes
AND Success message is displayed
```

**Scenario 2.3: QR Scan with Token Only**
```
GIVEN QR scanner modal is open
WHEN user scans QR code containing invitation token only (no embedded config)
THEN scanner modal closes
AND User is redirected to invitation acceptance flow
```

**Scenario 2.4: Invalid QR Code Handling**
```
GIVEN QR scanner modal is open
WHEN user scans invalid QR code (not invitation format)
THEN error message "Invalid QR code. Please scan a valid invitation QR code." is displayed
AND scanner continues to operate for retry
AND modal remains open
```

**Verification Method:**
- Manual testing: Scan various QR codes (valid, invalid, different formats)
- Automated: Unit tests for decodeConfigFromInvite with various inputs
- Integration: Test with actual QR codes generated by the system

---

### AC3: Camera Permission Handling

**Priority:** High

**Scenario 3.1: Camera Permission Granted**
```
GIVEN user has not previously granted camera permission
WHEN user clicks "Scan QR Code" button
THEN browser permission prompt is displayed
AND upon granting, camera preview activates
AND QR scanning becomes available
```

**Scenario 3.2: Camera Permission Denied**
```
GIVEN user clicks "Scan QR Code" button
WHEN user denies camera permission
THEN error message explains camera requirement
AND Manual input option remains available
AND "Try Again" button is displayed
```

**Scenario 3.3: No Camera Available**
```
GIVEN device does not have camera hardware
WHEN user navigates to config setup page
THEN "Scan QR Code" button is hidden or disabled
AND Manual input is the only option
```

**Verification Method:**
- Manual testing: Test permission flow on different browsers
- Device testing: Test on device without camera
- Regression: Ensure manual input always works

---

### AC4: Error Handling and Edge Cases

**Priority:** Medium

**Scenario 4.1: HTTPS Requirement Detection**
```
GIVEN application is accessed via HTTP (not localhost)
WHEN user attempts to open QR scanner
THEN warning message explains HTTPS requirement
AND QR scan option is disabled
AND Manual input remains available
```

**Scenario 4.2: Config Validation Failure**
```
GIVEN user scans QR code with invalid Supabase config
WHEN config is extracted and validated
THEN validation errors are displayed
AND Form fields are not auto-filled
AND User can manually correct the config
```

**Scenario 4.3: Camera Stream Error**
```
GIVEN QR scanner modal is open
WHEN camera stream fails (hardware or browser issue)
THEN error message explains the issue
AND Troubleshooting steps are provided
AND Manual input option is emphasized
```

**Scenario 4.4: Resource Cleanup**
```
GIVEN QR scanner modal is open with active camera
WHEN user closes modal
THEN camera stream stops immediately
AND Media device references are released
AND No memory leaks occur
```

**Verification Method:**
- Manual testing: Test each error scenario
- Browser DevTools: Verify camera stream cleanup
- Memory profiling: Check for leaks with repeated open/close

---

### AC5: Mobile Compatibility

**Priority:** Medium

**Scenario 5.1: iOS Safari Compatibility**
```
GIVEN user is on iOS Safari
WHEN user opens QR scanner
THEN camera starts with user gesture requirement
AND Scanner works in portrait and landscape
AND Touch targets are at least 44x44 pixels
```

**Scenario 5.2: Android Chrome Compatibility**
```
GIVEN user is on Android Chrome
WHEN user opens QR scanner
THEN camera starts with user gesture requirement
AND Scanner works in portrait and landscape
AND Back button behavior is correct
```

**Scenario 5.3: Responsive Camera Preview**
```
GIVEN user is on mobile device
WHEN QR scanner is open
THEN camera preview fills available width
AND UI elements are positioned for mobile screens
AND Orientation changes are handled gracefully
```

**Verification Method:**
- Device testing: Test on actual iOS and Android devices
- Responsive design: Test various screen sizes
- Orientation: Test portrait/landscape transitions

---

### AC6: Dark Mode Support

**Priority:** Medium

**Scenario 6.1: QR Code Display Dark Mode**
```
GIVEN user has dark mode enabled
WHEN QR code is displayed in invite modal
THEN QR code has appropriate contrast
AND Modal background uses dark theme colors
AND All text is readable in dark mode
```

**Scenario 6.2: QR Scanner Modal Dark Mode**
```
GIVEN user has dark mode enabled
WHEN QR scanner modal is open
THEN Camera preview border is visible
AND Modal background uses dark theme colors
AND Error messages are readable in dark mode
```

**Verification Method:**
- Visual testing: Switch between light/dark modes
- Contrast check: Use contrast checker for accessibility
- Screenshot comparison: Verify both themes look correct

**Priority:** Low

**Scenario 7.1: English Translations**
```
GIVEN user has English language selected
WHEN QR-related UI is displayed
THEN all text is in English
AND no translation keys are displayed
```

### AC8: Accessibility

**Priority:** Medium

**Scenario 8.1: Keyboard Navigation**
```
GIVEN user is using keyboard only
WHEN QR-related UI is displayed
THEN all buttons are accessible via Tab key
AND Enter/Space activates buttons
AND Focus is visible on all interactive elements
```

**Scenario 8.2: Screen Reader Support**
```
GIVEN user is using screen reader
WHEN QR scanner opens
THEN Modal purpose is announced
AND Camera status is announced
AND Scan success/error is announced
AND Error messages are associated with controls
```

**Scenario 8.3: Focus Management**
```
GIVEN QR scanner modal is closed
WHEN modal closes
THEN Focus returns to trigger button
AND Focus trap is released
```

**Verification Method:**
- Keyboard testing: Navigate all QR features with keyboard only
- Screen reader: Test with VoiceOver (iOS) and NVDA (Windows)
- Focus visual: Verify focus indicators are visible

---

### AC9: Manual Input Fallback

**Priority:** High

**Scenario 9.1: Manual Input Always Available**
```
GIVEN QR scanning is unavailable (permission denied, no camera, etc.)
WHEN user is on config setup page
THEN manual input form is fully functional
AND Config can be entered manually
AND Submit works without QR scanning
```

**Scenario 9.2: Manual Correction After Scan**
```
GIVEN config was auto-filled from QR scan
WHEN user modifies form fields
THEN modifications are accepted
AND Validation applies to modified values
```

**Verification Method:**
- Functional testing: Complete full setup without QR scanning
- Edit testing: Modify auto-filled values and verify submission

---

### AC10: Performance

**Priority:** Low

**Scenario 10.1: QR Generation Speed**
```
GIVEN invite link is generated
WHEN "Show QR Code" is clicked
THEN QR code renders within 500 milliseconds
AND UI remains responsive during generation
```

**Scenario 10.2: QR Scanning Speed**
```
GIVEN QR scanner is active
WHEN valid QR code is positioned in frame
THEN QR code is detected within 2 seconds
AND Config is extracted and validated within 3 seconds
```

**Scenario 10.3: Camera Performance**
```
GIVEN QR scanner is active
THEN camera preview maintains 30fps minimum
AND UI remains responsive during scanning
```

**Verification Method:**
- Performance testing: Measure rendering and detection times
- Frame rate: Monitor camera preview performance
- Memory: Check for memory leaks during extended use
